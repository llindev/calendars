# Calendars

Public iCalendar (`.ics`) subscriptions hosted via GitHub Pages, auto-generated by GitHub Actions.

## Subscribe to a calendar

Replace `<your-username>` and `<repo-name>` with your actual GitHub username and repository name.

| Calendar | Subscription URL |
|---|---|
| Weekly Team Sync | `https://<your-username>.github.io/<repo-name>/weekly-meeting.ics` |
| Monthly Review | `https://<your-username>.github.io/<repo-name>/monthly-review.ics` |

**How to subscribe:**

- **Google Calendar** → _Other calendars_ → _+ (Add)_ → _From URL_ → paste the URL above
- **Apple Calendar** → _File_ → _New Calendar Subscription_ → paste the URL
- **Outlook** → _Add calendar_ → _Subscribe from web_ → paste the URL
- **Thunderbird** → _New Calendar_ → _On the Network_ → paste the URL

## How it works

```
scripts/
├── base.py               # Base class – subclass this for each calendar
├── example_weekly.py     # Recurring weekly event (every Monday)
└── example_monthly.py    # Recurring monthly event (first Friday)

generate.py               # Discovers & runs all generators → writes output/
output/                   # Generated .ics files served by GitHub Pages
.github/workflows/
├── generate-calendars.yml  # Runs generate.py daily + on push, commits output/
└── pages.yml               # Deploys output/ to GitHub Pages
```

The `generate-calendars.yml` workflow runs:
- **On schedule** – every day at 02:00 UTC (keeps events rolling forward)
- **On push to `main`** – whenever scripts or `generate.py` change
- **Manually** – via _Actions → Generate Calendars → Run workflow_

## Adding a new calendar

1. Create a new file in `scripts/`, e.g. `scripts/my_calendar.py`.
2. Subclass `CalendarGenerator` from `scripts.base`:

```python
from datetime import datetime, timedelta, timezone
import uuid
from icalendar import Calendar, Event
from scripts.base import CalendarGenerator

class MyCalendar(CalendarGenerator):
    @property
    def calendar_id(self) -> str:
        return "my-calendar"          # → output/my-calendar.ics

    @property
    def calendar_name(self) -> str:
        return "My Calendar"

    @property
    def calendar_description(self) -> str:
        return "A short description."

    def generate(self, cal: Calendar) -> None:
        event = Event()
        event.add("uid", str(uuid.uuid4()))
        event.add("summary", "My Event")
        event.add("dtstart", datetime(2026, 3, 1, 9, 0, tzinfo=timezone.utc))
        event.add("dtend",   datetime(2026, 3, 1, 10, 0, tzinfo=timezone.utc))
        event.add("dtstamp", datetime.now(timezone.utc))
        cal.add_component(event)
```

3. Push to `main`. GitHub Actions will generate the `.ics` file and publish it automatically.

## Local development

```bash
pip install -r requirements.txt
python generate.py
# → output/weekly-meeting.ics
# → output/monthly-review.ics
```

## GitHub Pages setup (one-time)

1. Go to **Settings → Pages** in your repository.
2. Under **Source**, select **GitHub Actions**.
3. Push to `main` to trigger the first deployment.
